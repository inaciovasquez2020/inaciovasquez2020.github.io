Technical Correspondence

Repository-native correspondence for research artifacts, manuscript questions, audit requests, verification issues, and standards inquiries.

Priority Rule: Requests containing a concrete repository identifier, commit, pull request, registry record, or DOI receive highest priority.

Include at Least One Identifier

  • Repository name
  • Branch, commit hash (e.g., 606f147), or pull request number (e.g., #282)
  • Module or theorem label
  • Registry record, bridge record, or field code
  • DOI, manuscript code, or audit protocol identifier

Inquiry Channels

  • Institutional and editorial: manuscripts, submission history, and publication metadata.
  • Verification and audit: urf-verifier, scientific-infrastructure, and repository-native reproducibility checks.
  • Formalization and frontier: flagship-lean, CorrRank, final-wall-fo-k-locality, and cycle-local-rigidity.
  • Cross-field mapping and registry: urf-core community surfaces, URF-11 field records, and translation-rule records.
  • Website and index corrections: vasquez-index, inaciovasquez2020.github.io, ORCID, and DOI updates.
